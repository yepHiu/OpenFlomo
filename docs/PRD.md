# OpenFlomo 产品需求文档（PRD）

> 基于 Flomo 浮墨笔记产品理念，为 OpenFlomo 项目定义功能范围与优先级。

**文档版本**：v1.0  
**更新日期**：2025-03-03

---

## 一、产品概述

### 1.1 产品定位

OpenFlomo 是一款轻量级、碎片化记录与灵感管理工具，核心理念为「先记录，后整理」，降低记录门槛，让想法被快速捕捉并在回顾中产生联系。

### 1.2 目标用户

| 用户类型 | 描述 |
|---------|------|
| 备忘录用户 | 需要跨平台、快速同步的随手记 |
| 日记/便签用户 | 记录想法与情绪，轻量表达 |
| 知识记录用户 | 有工作经验，希望持续积累知识与生活记录 |

### 1.3 核心痛点

- 传统笔记软件过重，打开与编辑负担大
- 记录时易被排版、标题、分类分散注意力
- 已记录内容很少被回顾，难以形成「个人上下文」

## 二、优先级说明

| 优先级 | 含义 | 发布阶段建议 |
|--------|------|--------------|
| **P0** | 必须有，核心价值，无则产品不成立 | MVP / 首版 |
| **P1** | 重要，显著提升体验与留存 | V1.x |
| **P2** | 有价值，增强粘性与差异化 | V2.x |
| **P3** | 可选/长期，锦上添花或商业化 | 后续迭代 |

## 三、功能列表与优先级

### P0 — 核心必选（MVP）

| 编号 | 功能模块 | 功能描述 | 验收要点 |
|------|----------|----------|----------|
| P0-1 | **极简记录** | 单条输入、无标题、无复杂格式，点击即记 | 打开即可输入、发送即保存，无必填项 |
| P0-2 | **卡片列表** | 按时间倒序展示所有 memo，单条即一卡 | 列表可滚动、单条可点击查看/编辑 |
| P0-3 | **多级标签** | 支持 `#父标签/子标签` 形式组织内容 | 输入时支持 # 唤起标签、列表可按标签筛选 |
| P0-4 | **标签筛选与搜索** | 按标签查看 memo，支持关键词搜索 | 筛选结果正确、搜索可匹配正文与标签 |
| P0-5 | **数据持久化** | 记录本地或服务端可靠存储、可读可写 | 重启/刷新后数据不丢失、支持基础 CRUD |

**P0 完成标准**：用户能「打开 → 写一条带标签的 memo → 在列表中看到并按标签/搜索找到」即视为 MVP 成立。

---

### P1 — 重要功能（V1.x）

| 编号 | 功能模块 | 功能描述 | 验收要点 |
|------|----------|----------|----------|
| P1-1 | **多端/多入口输入** | 除主应用外，支持 API、Web 剪藏等入口写入 | 通过 API 可创建 memo，并出现在主列表 |
| P1-2 | **每日回顾** | 按标签/时间筛选内容，支持定时或手动「回顾」推送 | 可配置回顾条件与时间，回顾列表可浏览 |
| P1-3 | **热力图** | 按日展示记录数量，颜色深浅表示活跃度 | 日历热力图正确反映每日条数 |
| P1-4 | **基础统计** | 总条数、按标签统计、按日/周/月统计 | 统计数字与真实数据一致 |
| P1-5 | **单条编辑与删除** | 对已有 memo 可编辑内容与标签、可删除 | 编辑后即时生效，删除可确认防误触 |
| P1-6 | **账号与多端同步** | 用户账号体系，数据在 Web/App 间同步 | 登录后各端看到同一份数据 |

**P1 完成标准**：记录、组织、回顾、统计形成闭环，多端可写可看，具备「可长期使用」的完整体验。

---

### P2 — 增强功能（V2.x）

| 编号 | 功能模块 | 功能描述 | 验收要点 |
|------|----------|----------|----------|
| P2-1 | **批注与引用** | 在一条 memo 中引用/链接其他 memo，形成关联 | 可@或链接到其他条，点击可跳转 |
| P2-2 | **随机漫步** | 基于标签/内容推荐「相关」或随机 memo 供回顾 | 有随机或推荐入口，结果可点击进入详情 |
| P2-3 | **历史版本** | 单条 memo 支持查看与恢复历史版本 | 有历史版本列表，可对比、可恢复 |
| P2-4 | **导出** | 支持导出为 JSON/Markdown 等，便于备份与迁移 | 导出内容完整、格式可用 |
| P2-5 | **分享** | 单条或组合生成分享链接/卡片图 | 分享后他人可读（只读），样式清晰 |
| P2-6 | **快捷输入与快捷键** | 全局快捷键或快捷入口快速打开输入框 | 快捷键可配置，输入框可快速调起 |

**P2 完成标准**：回顾与关联能力增强，数据可导出可分享，产品具备明显差异化与安全感。

---

### P3 — 可选/长期（后续迭代）

| 编号 | 功能模块 | 功能描述 | 验收要点 |
|------|----------|----------|----------|
| P3-1 | **语音输入** | 语音转文字后作为 memo 内容，可自动去语气词等 | 语音识别准确、结果可编辑后保存 |
| P3-2 | **相关笔记（语义）** | 基于内容语义推荐相关 memo，不限于标签 | 推荐结果与当前条语义相关 |
| P3-3 | **AI 洞察/报告** | 基于记录生成周期洞察或思维模式报告 | 报告可读、有参考价值 |
| P3-4 | **微信/第三方快捷输入** | 通过微信服务号、Telegram 等快速发送为 memo | 从第三方发送后可在主应用看到 |
| P3-5 | **主题与个性化** | 深色模式、字体大小、布局偏好等 | 设置生效且持久化 |
| P3-6 | **高级统计与可视化** | 标签趋势、词云、时间分布图等 | 图表正确、性能可接受 |

**P3 完成标准**：在 P0～P2 稳定后，按资源与战略选择实现，用于体验提升或商业化探索。

## 四、非功能需求

| 类型 | 要求 |
|------|------|
| **性能** | 首屏与输入响应快，列表滚动流畅，符合「轻」的定位 |
| **可用性** | 新用户无需教程即可完成「记一条、加标签、查一条」 |
| **数据** | 数据可备份、可导出，敏感信息考虑端到端或本地优先方案 |
| **多端** | Web 优先，后续可扩展桌面/移动端，数据与账号一致 |

---

## 五、迭代建议

1. **第一阶段**：只做 P0，交付可用的 MVP，验证「记录 + 标签 + 列表 + 搜索」闭环。
2. **第二阶段**：完成 P1，补全回顾、统计、多端同步与账号体系。
3. **第三阶段**：按需实现 P2，强化回顾、关联、导出与分享。
4. **第四阶段**：根据用户反馈与资源，选择性落地 P3（如语音、AI、微信入口等）。
